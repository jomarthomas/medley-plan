
# GitHub Issues (actionable, prioritized)

Create the following Issues in the repo tracker. Keep titles concise, paste the descriptions into each issue body, and apply the suggested labels.


1) Initial documentation setup
- Description: Finalize `README.md`, `PLANS.md`, and core files in `/docs/` to match the medley-plan specification (device list, date-based filenames, update cadence).
- Labels: admin, strategy

2) Health-data folder setup
- Description: Verify `/health-data/` subfolders exist and confirm accepted file types and date-based naming in `/health-data/README.md`.
- Labels: health, admin

3) First monthly review execution
- Description: Run the first 30-minute monthly review and record the Health Signal Review observations and snapshot tag in `PLANS.md`.
- Labels: review

4) First quarterly checkpoint
- Description: Schedule and execute the first quarterly fitness checkpoint (5K TT or 20-min TT); log results in `PLANS.md`.
- Labels: training, review

5) First race plan draft
- Description: Finalize concise execution guidance in `docs/race-plans.md` for 5K, 10K, and Half.
- Labels: race, training

6) Polar H10 integration guidance
- Description: Document chest-strap usage for VO2 and tempo sessions, and note data export expectations (CSV) for Polar H10.
- Labels: training, health

7) HR zone calibration review
- Description: Establish a short HR zone calibration procedure (owner, protocol) and validate with Polar H10 sessions.
- Labels: tracking, training

8) Privacy & repo visibility confirmation
- Description: Confirm the repository visibility is private and document the access policy for health exports.
- Labels: admin

9) Device export sample upload
- Description: Upload one representative export per device (Apple Watch Ultra, Polar H10, KardiaMobile 6L, OMRON, scale) to `/health-data/` to validate naming and storage.
- Labels: health, tracking

10) Snapshot tagging procedure
- Description: Document tagging steps and changelog format for monthly snapshots; assign owner for monthly tags.
- Labels: admin, review

Notes
- Keep Issues focused and actionable; keep the total under 12. Close each when the corresponding doc or task is confirmed.
